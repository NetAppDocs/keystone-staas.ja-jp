= Keystoneのパフォーマンスサービスレベル
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Keystone STaaSは、ストレージの種類ごとに、事前に定義されたパフォーマンス サービス レベル（PSL）を提供します。各PSLは、パフォーマンス目標（IOPS、スループット、レイテンシ）、サポートされるプロトコル、プラットフォーム、および最小容量要件を規定しています。利用可能なPSLを確認し、ワークロードに適したサービス レベルを選択して、契約プランを立ててください。

Keystoneサブスクリプションには、1つまたは複数のストレージタイプにわたる複数のPSLを含めることができます。各PSLには、契約期間中にそのサービスレベルで請求される最小容量である、確約容量が設定されています。


NOTE: AFXサブスクリプションはAFXストレージタイプに限定されており、現在提供されているPSLはExtremeのみです。



== パフォーマンス サービス レベル インスタンス

パフォーマンス サービス レベル（PSL）は、環境内に複数のインスタンス（パフォーマンス サービス レベル インスタンス（PSLI）と呼ばれる）を持つことができます。各PSLIは、そのパフォーマンス サービス レベルに割り当てられた個別のストレージ システムであり、コミットされた容量はPSLIごとに測定されます。

* 統合ストレージまたはブロック最適化ストレージの場合、PSLIは、内部または外部に接続されたストレージ メディアを備えたHAペアの2つのストレージ コントローラで構成されるストレージ アレイです。
* AFXストレージの場合、PSLIは単一のAFXストレージコントローラです。AFXは、コントローラ（演算処理）とドライブシェルフ（ストレージ）が独立しており、それぞれ個別に拡張できる分離型アーキテクチャを採用しています。IOPSとスループットの目標値はコントローラごとに適用されます。コミット済み容量はAFXクラスタレベルで測定され、クラスタ内のすべてのコントローラとドライブシェルフが含まれます。


IOPS、スループット（GBps）、レイテンシ（ms）などのパフォーマンス指標は、PSLI ごとに測定されます。


TIP: NetApp Keystone STaaSパフォーマンスサービスレベルの詳細なサービス説明は入手可能です。  https://www.netapp.com/services/keystone/terms-and-conditions/["こちらをご覧ください"^] 。

サポートされているデータ ストレージ タイプ（ユニファイド、ブロック最適化、AFX、オブジェクト、クラウド サービス）の基本パフォーマンス サービス レベルについては、以下のセクションで説明します：

* ^1^相互に排他的な目標。実際のパフォーマンスは、オペレーティング システムのバージョン、ハードウェア、ワークロードの種類、同時操作の数など、さまざまな要因によって異なる場合があります。

== 統合ストレージ、ブロック最適化ストレージ、およびAFAストレージのパフォーマンス測定条件

基本パフォーマンス サービス レベル メトリックは、次の条件によって異なります。

* パフォーマンス サービス レベルは ONTAP 9.8 以降をサポートします。
* 統合ストレージの場合、
+
** *IOPS*: NFS を使用した ONTAP 9.16.1 の場合、各パフォーマンス レベルのインスタンスでは、読み取り比率 70%、書き込み比率 30%、ブロック サイズ 8 KB、レイテンシ 1 ミリ秒 (標準の場合は 4 ミリ秒) のランダム アクセスがサポートされます。
** *スループット*: NFS を使用した ONTAP 9.16.1 の場合、各パフォーマンス レベルのインスタンスでは、100% 読み取りと 32 KB のブロック サイズによるシーケンシャル アクセスがサポートされます。


* ブロック最適化ストレージの場合、
+
** *IOPS*: FCP を搭載した ONTAP 9.16.1 の場合、各パフォーマンス レベルのインスタンスでは、読み取り比率 70%、書き込み比率 30%、ブロック サイズ 8 KB、レイテンシ 1 ミリ秒のランダム アクセスがサポートされます。
** *スループット*: FCP を搭載した ONTAP 9.16.1 の場合、各パフォーマンス レベルのインスタンスは、100% 読み取りと 64 KB のブロック サイズによるシーケンシャル アクセスをサポートします。


* AFX ストレージの場合、
+
** *IOPS*：ONTAP 9.18.1 with NFSv4では、各パフォーマンス レベル インスタンスは、読み取り70%、書き込み30%の比率でランダム アクセスをサポートし、ブロック サイズは4KBです。
** *スループット*：ONTAP 9.18.1 with NFSv4では、各パフォーマンス レベル インスタンスは、100%読み取りと64 KBのブロック サイズによるシーケンシャル アクセスをサポートします。


* レイテンシには、次の要素は含まれません。
+
** アプリケーションまたはホストのレイテンシ
** コントローラポートとの間のお客様のネットワークレイテンシ
** FabricPool の場合のオブジェクトストアへのデータ転送に関連するオーバーヘッド


* レイテンシの値は、MetroCluster 書き込み処理には適用されません。これらの書き込み処理は、リモートシステムの距離に依存します。
* 想定IOPS_は、階層化ポリシーが「none」に設定されていて、ブロックがクラウドにない場合にのみFabricPool のターゲットになります。想定IOPS_は、SnapMirror同期関係にないボリュームを対象としています。
