= Keystoneの請求について学ぶ
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
NetApp Keystone STaaSでは、コミットした分の料金を支払い、それを超えて使用した分についてのみ追加料金が発生します。このページでは、従量課金制モデルの仕組み（ストレージコミットメントの価格設定方法、容量の測定と請求方法、請求書の発行時期など）について説明します。



== Keystone 価格設定モデル

NetApp Keystoneサブスクリプションなら、コストを予測できるだけでなく、ワークロードの増加に応じて柔軟に規模を拡大できます。NetApp Keystone料金は、お客様が選択されたパフォーマンスサービスレベルと、お客様がコミットされた容量に基づいて決定されます。

Keystoneでは、次の請求機能を提供しています。

* さまざまなワークロードのニーズに合わせて、IOPS とレイテンシーのコミット容量に基づいて料金を支払うことができます。さまざまなパフォーマンス サービス ティア (Extreme、Premium、Standard、Value、Object、 Cloud Volumes ONTAP)により、購入したパフォーマンス サービス レベルに基づいてストレージを管理できます。
* また、容量の利用状況に応じて、コミット済み容量と従量課金制の容量を予測可能で提示します。
* ハードウェア、コア OS 、サポートのバンドル価格を 1TiB 価格で選択できます。ストレージ タイプ (統合、ブロック、最適化、オブジェクト、クラウド ストレージ サービス) ごとに 1 つの請求書が発行されます。
* サービス期間と支払いオプションは、月ごと、四半期ごと、半年ごと、年ごとなど、柔軟に選択できます。


Keystone請求は、コミット済み容量と可変バースト消費に基づいて行われます。

Keystoneでサポートされるさまざまな容量については、link:../concepts/supported-storage-capacity.html["Keystoneのストレージ容量タイプ"]を参照してください。



== Keystoneが容量を測定する方法

Keystoneでは、請求額の計算に3つの容量概念を使用します。それぞれを理解することで、予期しない料金の発生を回避できます。



=== コミット済み容量

コミット容量とは、Keystoneサブスクリプションに契約する際に、特定のパフォーマンスサービスレベルに対して購入することに同意した容量のことです。この金額はKeystoneオーダーで固定され、ワークロードが実際に使用する容量に関わらず、各請求期間ごとに全額が請求されます。

例えば、Premiumサービスレベルで100TiBの容量を契約した場合、その月の実際の使用量が80TiBであっても、100TiB分の料金が請求されます。

コミット容量は、お客様とNetAppまたはパートナーとの合意に基づき、単一のサブスクリプション内で複数のパフォーマンスサービスレベルにまたがることができます。

コミット済み容量を確認し、実際の使用状況を追跡するには、link:../integrations/current-usage-tab.html["現在の消費量の詳細を表示する"]を参照してください。



=== 使用済み容量

消費容量とは、ワークロードが実際に任意の時点で使用している容量のことです。Keystoneは請求期間全体を通して、お客様の消費量を継続的に監視および測定します。少なくとも5分ごとに、お客様のサブスクリプションにおける現在の消費容量を記録する消費量記録が生成されます。これらの記録は請求期間全体にわたって集計され、請求書と利用状況レポートが作成されます。

5分間隔の監視により、きめ細かな追跡が可能となり、バースト使用が正確に検出され、請求されます。期間の終わりに推定されることはありません。

消費容量が契約容量以下である場合、契約容量に基づく料金のみが適用されます。消費容量が契約容量を超えた場合、バースト課金が適用されます。



=== バースト消費

バースト消費とは、契約容量を超えて使用する容量のことです。バースト制限は、Keystoneの契約で決定され、指定されます。デフォルトでは、契約容量の20%増に設定されています。また、バースト容量の上限を、契約容量の40%または60%に設定することもできます。詳細については、link:../concepts/burst-capacity.html["バースト容量オプション"]を参照してください。

例えば、契約容量が100 TiBで、115 TiBを使用した場合、超過分の15 TiBはバーストレートで課金されます。

消費された容量が、パフォーマンスサービスレベルに対して約束された容量を超えた場合、バースト消費が記録され、それに応じて課金されます。バースト容量制限を超える使用は、"Above burst limit"として表示されます。

バースト消費量は、請求期間中に生成された各消費記録ごとに計算されます。これは、契約済みの容量に加えて、超過消費した容量の量と期間の両方を反映しています。サブスクリプションのバースト消費傾向を確認するには、link:../integrations/consumption-tab.html["Keystoneサブスクリプションの消費傾向を表示する"]を参照してください。



== 請求スケジュール

請求頻度は、Keystoneの契約で設定されます。以下の表は、各オプションにおける、コミットメント容量とバースト使用量に対する請求書の発行時期を示しています。

[cols="1,2,2"]
|===
| 請求頻度 | コミット容量の請求書 | バースト使用量の請求書 


| 毎月 | 翌月に送付 | 同じ請求書に含まれる 


| 四半期 | 各四半期の開始時に送信されます | 各四半期の終わりに送信 


| 半年ごと | 6ヶ月ごとの開始時に送付されます | 各四半期の終わりに送信 


| 毎年 | 各サブスクリプション年度の開始時に送信されます | 各四半期の終わりに送信 
|===

NOTE: 四半期払い、半年払い、年払いの場合、サブスクリプション期間中にコミット済み容量に変更が生じた場合は、そのサブスクリプション年度の残りの期間に対する日割り計算された最低支払額の請求書が同日に送付されます。請求は、コミット済み容量の変更が有効になった日から計算されます。
