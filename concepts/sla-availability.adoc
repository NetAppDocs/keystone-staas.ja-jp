= Keystoneの可用性 SLO
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
可用性サービスレベル目標（SLO）は、統合ストレージおよびブロック最適化ストレージの場合、パフォーマンスサービスレベルインスタンス（PSLI）あたり、またはAFXストレージコントローラあたり、99.999%のサービス可用性を目標としています。詳細には、可用性の測定方法、含まれるストレージの種類とパフォーマンスサービスレベル、目標が達成されなかった場合のサービスクレジットなどが含まれます。



== 指標

* *月間稼働率* = [(1ヶ月間の有効秒数 - その月のPSLIのサービス停止秒数) / 1ヶ月間の有効秒数] × 100%
* *サービス利用不可*：監視ツールによって報告される、PSLIによってデータ読み取りまたは書き込み操作が処理されない期間。NetAppと合意した計画的なメンテナンス、アップグレード、またはサポート活動により、またはNetAppの管理または責任の範囲外の状況により、STaaSサービスが利用できない時間は除外されます。
* *対象となる秒数*：上記で定義されたサービス停止期間を除き、稼働時間計算にカウントされる月間の秒数。

=== サービスクレジットの計算

サービスクレジットは、次の式を使用して決定されます。

サービスクレジット=（影響を受ける容量/コミット済み容量の合計）X容量料金Xクレジットの割合

ここで、

* *影響を受ける容量*：影響を受ける格納容量。
* *合計コミット容量*: Keystone 注文のパフォーマンス サービス レベルのコミット容量。
* *容量料金*: 当月の影響を受けるパフォーマンス サービス レベルの料金。
* *クレジットパーセンテージ*：サービスクレジットの事前定義されたパーセンテージ。


* 例 *

次の例は、サービスクレジットの計算方法を示しています。

. 月単位のアップタイムを計算して、サービスクレジットの割合を決定します。
+
** 30日間の対象秒数：30（日）X 24（時間/日）X 60（分/時間）X 60（秒/分）= 2、592、000秒
** サービス停止時間（秒）：95秒
+
次の式を使用します。月間アップタイムの割合=[（2、592、000-95）/（2、592、000）] X 100

+
計算に基づくと、月間稼働率は99.996%、サービスクレジット率は5%です。



. サービスクレジットの計算：
+
|===


| * サービスレベル * | *影響を受ける容量* | *コミット済み容量の合計* | *容量料金* | *クレジットの割合* 


 a| 
最高レベル
| 95秒で10TiB | 100TiB | 1、000ドル | 5% 
|===
+
式を使用：サービスクレジット=（10/100）X 1000 x 0.05

+
計算に基づくと、サービス クレジットは$5です。





=== サービスクレジットリクエスト

SLA違反が検出された場合は、NetApp Keystoneサポートにプライオリティ3（P3）サポートチケットをオープンします。

* 次の詳細情報が必要です。
+
.. Keystoneサブスクリプション番号
.. ボリュームとストレージコントローラの詳細
.. 問題のサイト、時刻、日付、および説明
.. レイテンシ検出の計算時間
.. 測定ツールと測定方法
.. その他の該当する文書


* NetApp Keystoneサポートで開いたP3チケットについて、以下に示すようにExcelシートに詳細を入力してください。
+
image:sla-breach.png["Excelシートテンプレート"]



[NOTE]
====
* サービスクレジットリクエストは、NetApp Keystoneサポートが違反を確認してから6週間以内に開始する必要があります。すべてのサービスクレジットがNetAppによって承認され、承認される必要があります。
* サービスクレジットは、今後の請求書に適用される場合があります。サービスクレジットは、期限切れのKeystoneサブスクリプションには適用されません。NetAppサポートに連絡してサービスクレジットリクエストに必要なサポートケースを開くには、link:../concepts/support-services.html["Keystoneに関するサポートを受ける"]を参照してください。
